Privacy Policy — Zonov Claim Agent

Last Updated: May 8, 2026  |  Version 1.0  |  Applies to Chrome Extension v1.0.0+

This Privacy Policy describes how Zonov AI ("we", "us") collects, uses, and protects information through the Zonov Claim Agent Chrome Extension ("Extension"). By installing and using the Extension, you agree to the practices described in this policy.

Who uses this Extension? This Extension is intended exclusively for authorised hospital staff (claim executives, doctors, nursing staff) at empanelled hospitals. It is not a public consumer product. All data processing is done on behalf of the hospital.

1. Information We Collect

The Extension collects only the minimum data required to perform its core functions:

Data Type What exactly Where stored Sent externally?
Patient identifiers Visit ID, UHID, Aadhaar number, patient name, insurance scheme — read from the hospital portal page Device only (chrome.storage.local) No
Medical documents PDF and image files uploaded by staff (Discharge Summary, OT Notes, Investigation Reports, etc.) In-memory only during session; optionally synced to hospital's Zonov account Only to Zonov backend
Document images (AI scan) Image data of uploaded documents sent for AI signature/stamp detection Not stored — processed in real-time Sent to Groq API
Staff profiles Doctor name, employee ID, category, phone, signature image, stamp image, photo Device (chrome.storage.local) + Zonov backend (S3) Only to Zonov backend
Hospital details Hospital name, registration number, address, phone, stamp image, logo Device (chrome.storage.local) + Zonov backend (S3) Only to Zonov backend
API credentials Auth token and license key for Zonov backend authentication Device only (chrome.storage.local) No
Export history Date, filename, scheme name of exported claim PDFs Device only (chrome.storage.local) No
Voice audio Microphone audio for voice navigation commands (hands-free use) Not stored — processed in real-time only Sent to Deepgram API

2. What We Do NOT Collect

  • Browsing history, URLs, or activity on any website other than the configured hospital portal
  • Passwords or login credentials of the hospital portal
  • Financial information such as credit/debit card numbers
  • Data from any page the Extension is not explicitly active on
  • Any data from children — this Extension is for healthcare professionals only

3. How We Use the Data

  • Display the correct document checklist based on the patient's insurance scheme (RGHS, MAA, PM-JAY, MCSBY, ESIC, CGHS, etc.)
  • Verify documents for completeness — checking for doctor signature, stamp, and hospital stamp
  • Apply doctor signatures and stamps on claim documents
  • Generate and export complete claim PDF packages for hospital insurance submission
  • Sync staff profiles and hospital data across devices within the same hospital account
  • Show daily summary and dashboard statistics for the claim team

4. Third-Party Services Used

Service Provider Purpose Data Sent Their Privacy Policy

Zonov Backend Zonov AI (apiprod.zonov.ai) Store staff profiles, hospital data, and patient document metadata for the hospital account Staff profiles, signatures, stamps, document URLs — per hospital account agreement This policy

We do not sell, rent, or trade any data to advertisers or unrelated third parties.

6. Data Storage & Retention

  • Local storage (chrome.storage.local): Data persists on the device until the Extension is uninstalled or the user manually clears it from the Extension settings.
  • Zonov backend (AWS S3 + database): Staff and hospital data is retained per the service agreement with the hospital. Hospitals can request full data deletion at any time.
  • Groq / Deepgram: We do not store any data on their servers. Groq and Deepgram process requests in real-time per their own retention policies.

7. Security

  • All communication with Zonov backend and third-party APIs uses HTTPS encryption
  • API credentials are stored in chrome.storage.local — not in JavaScript source code, cookies, or localStorage
  • Document images are processed as temporary in-memory data URLs and are not persisted beyond the session
  • The Extension only activates on the hospital portal domains configured in the manifest

8. Your Rights

Authorised hospital staff and hospital administrators may:

  • View, edit, or delete staff profiles directly from within the Extension
  • Clear all local data by uninstalling the Extension
  • Request complete data deletion from Zonov backend by contacting us at saurabh@zonov.ai
  • Request a copy of all data stored for their hospital account

9. Changes to This Policy

We may update this policy to reflect changes in the Extension's functionality or applicable regulations. The "Last Updated" date at the top will always reflect the most recent version. Continued use of the Extension after changes constitutes acceptance of the updated policy.
